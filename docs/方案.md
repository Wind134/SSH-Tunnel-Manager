## 1. 项目概述

### 1.1 背景

国内云服务器（如腾讯云）访问 GitHub 等海外资源不稳定。常见的解决方式是在本机（能翻墙）建立 SSH 反向隧道，将本机代理端口映射到远程机器，远程机器通过该端口走本机代理出去。目前这一过程完全依赖命令行，每次使用需要手动敲 `ssh -R` 命令，配置无法持久化，操作体验差。

### 1.2 目标

开发一个 Windows 桌面 GUI 工具，用于管理 SSH 反向隧道配置，实现一键启动/停止隧道，配置持久化存储，IP 地址加密保存。

### 1.3 核心用户场景

1. 添加一台远程服务器配置（IP、端口、用户名、密钥）
2. 启动隧道：本机代理端口 → SSH 隧道 → 远程机器端口
3. 监控隧道状态（连接/断开/重连）
4. 关闭隧道
5. 再次启动时无需重新配置

## 2. 技术选型

| 层 | 技术 | 选择理由 |
|---|---|---|
| **UI 框架** | WPF (.NET 8) | Windows 原生桌面框架，支持 XAML 声明式 UI，与 Windows 11 Fluent Design 深度集成 |
| **UI 美化** | ModernWpf | 开源 WinUI / Fluent 风格控件库，提供 Win11 原生感的控件模板、Mica 效果、圆角 |
| **运行时** | .NET 8.0-windows | 最新 LTS，支持 self-contained 发布，目标机器无需安装 runtime |
| **加密** | `System.Security.Cryptography.ProtectedData` | Windows DPAPI 原生 API，数据由当前用户凭据加密，无需管理密钥 |
| **SSH 进程管理** | `System.Diagnostics.Process` | 调用系统 OpenSSH（Win10/11 自带），管理其生命周期 |
| **配置存储** | JSON + DPAPI 字段加密 | 轻量、人类可读（IP 字段除外）、易于备份和迁移 |
| **目标框架** | net8.0-windows10.0.19041.0 | 最低支持 Windows 10 20H1，兼容 Windows 11 |

## 4. 数据模型

### 4.1 TunnelConfig（配置模型）

```csharp
public class TunnelConfig
{
    public Guid Id { get; set; }
    public string Name { get; set; }
    public int LocalPort { get; set; }          // 默认 7890
    public int RemotePort { get; set; }         // 默认 1080
    public string SshUser { get; set; }
    public string EncryptedHost { get; set; }   // DPAPI 加密后
    public int SshPort { get; set; }            // 默认 22
    public string KeyFilePath { get; set; }     // 可选
    public bool AutoReconnect { get; set; }
    public DateTime CreatedAt { get; set; }
    public DateTime ModifiedAt { get; set; }
}
```

### 4.2 配置存储格式（config.json）

```json
{
  "version": 1,
  "tunnels": [
    {
      "id": "a1b2c3d4-...",
      "name": "腾讯云-开发机",
      "localPort": 7890,
      "remotePort": 1080,
      "sshUser": "root",
      "encryptedHost": "AQAAANCMnd8...DPAPI加密数据...",
      "sshPort": 22,
      "keyFilePath": "",
      "autoReconnect": true,
      "createdAt": "2026-07-11T18:00:00",
      "modifiedAt": "2026-07-11T18:00:00"
    }
  ],
  "settings": {
    "autoStartMinimized": false
  }
}
```

### 4.3 安全性设计

| 要求 | 方案 |
|---|---|
| **IP 地址存储加密** | DPAPI `ProtectedData.Protect()` 加密，`Unprotect()` 解密。密钥绑定当前 Windows 用户 + 本机，config.json 泄露也无法解密 |
| **界面显示脱敏** | 列表页 IP 显示为 `123.45.***.***`，编辑时点击眼睛按钮才明文 |
| **SSH 私钥** | 仅存路径，不存私钥内容。安全由文件系统权限保证 |
| **日志安全** | 不记录密码、密钥内容 |

## 6. 核心流程

### 6.1 启动隧道流程

```
用户点击 [启动]
    │
    ▼
读取 TunnelConfig
    │
    ▼
解密 IP（DPAPI Unprotect）
    │
    ▼
检查本地端口是否被占用 ──是──→ 报错提示
    │
    否
    ▼
构造 SSH 命令
    │
    ▼
启动 Process
    │
    ▼
启动定时器监控进程 ── 进程退出 ──→ 判断 AutoReconnect
    │                              │
    ▼                              └──是──→ 等待 3s 后重连
更新状态 = 已连接                             最多重试 5 次
    │
    ▼
写入日志
```

### 6.2 SSH 命令构造

```
ssh -N -R 1080:127.0.0.1:7890 \
    -o ServerAliveInterval=30 \
    -o ServerAliveCountMax=3 \
    -o ExitOnForwardFailure=yes \
    root@123.45.67.89 -p 22 \
    [-i C:\Users\...\id_rsa]
```

### 6.3 自动重连策略

| 重连次数 | 等待时间 | 说明 |
|---|---|---|
| 1-3 次 | 3 秒 | 应对短暂网络抖动 |
| 4-5 次 | 10 秒 | 逐渐放慢频率 |
| 超过 5 次 | 停止 | 标记失败，由用户手动重试 |

## 10. 风险与注意事项

### 10.1 前置依赖

- 本机需有 OpenSSH Client（Win10 1809+ / Win11 默认已装）
- 远程机器需运行 SSH Server
- 远程机器的 `GatewayPorts no` 无需改动
- 本机需有代理客户端独立运行（Clash/v2ray 等）

### 10.2 异常处理

- SSH 进程异常退出 → 捕获退出码 + stderr → 显示用户可读的错误信息
- 本地端口被占用 → 启动前检测，提示用户更换
- 临时网络中断 → 自动重连（5 次上限）
- DPAPI 解密失败 → 温和提示，不崩溃
- 远程机器不可达 → 清晰的 connection refused 提示

### 10.3 已知限制

- 不支持多跳隧道
- 不支持 SSH 密码认证（仅密钥认证）
- 不内置代理 Client
