## 1. 项目概述

### 1.1 背景

国内云服务器（如腾讯云）访问 GitHub 等海外资源不稳定。常见的解决方式是在本机（能翻墙）建立 SSH 反向隧道，将本机代理端口映射到远程机器，远程机器通过该端口走本机代理出去。目前这一过程完全依赖命令行，每次使用需要手动敲 ssh -R 命令，配置无法持久化，操作体验差。

### 1.2 目标

开发一个 Windows 桌面 GUI 工具，用于管理 SSH 反向隧道配置，实现一键启动/停止隧道，配置持久化存储，敏感信息（IP、密码）加密保存，支持密码和密钥两种 SSH 认证方式。

### 1.3 核心用户场景

1. 添加一台远程服务器配置（IP、端口、用户名、认证方式）
2. 首次连接时确认 Host Key 指纹
3. 启动隧道：本机代理端口 -> SSH 隧道 -> 远程机器端口
4. 自动验证隧道连通性
5. 监控隧道状态（连接/断开/重连）
6. 关闭隧道，退出应用时优雅断开
7. 再次启动时无需重新配置

## 2. 技术选型

| 层 | 技术 | 选择理由 |
|---|---|---|
| **UI 框架** | WPF (.NET 8) | Windows 原生桌面框架，支持 XAML 声明式 UI |
| **UI 样式** | 原生 WPF 控件 + 手写 XAML | 未使用 ModernWpf 等第三方 UI 库；当前为浅色主题（白底深字），Mica / 主题跟随系统均未实现 |
| **运行时** | .NET 8.0-windows | 最新 LTS，支持 self-contained 发布，目标机器无需安装 runtime |
| **SSH 实现** | SSH.NET (Renci.SshNet) | 纯 .NET SSH 库，原生支持密码/密钥认证、端口转发、Host Key 校验。无需依赖系统 OpenSSH，无需外部进程管理 |
| **加密** | System.Security.Cryptography.ProtectedData | Windows DPAPI 原生 API，数据由当前用户凭据加密，无需管理密钥 |
| **配置存储** | JSON + DPAPI 字段加密 | 轻量、人类可读（敏感字段除外）、易于备份和迁移 |
| **目标框架** | net8.0-windows10.0.19041.0 | 最低支持 Windows 10 20H1，兼容 Windows 11 |

### 2.1 为什么选 SSH.NET 而不是 Process 调 OpenSSH

| 对比项 | Process + OpenSSH | SSH.NET |
|---|---|---|
| 密码认证 | Windows 上极难实现（需 sshpass 或 SSH_ASKPASS hack） | 原生支持，一行代码 |
| Host Key 校验 | 需解析 OpenSSH 输出或预写 known_hosts | 代码内比对指纹，可弹窗交互 |
| 端口转发 | 依赖 OpenSSH 的 -R 参数 | ForwardedPortRemote 类，in-process |
| 进程生命周期 | 需管理外部进程，崩溃/僵尸进程风险 | in-process 管理，应用退出自动清理 |
| 依赖 | 需系统装 OpenSSH Client | 仅一个 NuGet 包 |
| 调试 | 只能看 stderr/stdout | C# 异常，可直接断点 |

```csharp
public enum AuthMethod { Password, PrivateKey }
public enum HostKeyTrust { Unknown, Trusted, Rejected }

public class TunnelConfig
{
    public Guid Id { get; set; }

    public string HostKeyFingerprint { get; set; }   // SHA256 指纹
    public HostKeyTrust HostKeyTrust { get; set; }   // 信任状态
    public bool AutoReconnect { get; set; }
    public DateTime CreatedAt { get; set; }
    public DateTime ModifiedAt { get; set; }
}
```

### 4.3 配置文件管理

| 项目 | 说明 |
|---|---|
| **存储路径** | %APPDATA%\SSHTunnelManager\config.json（即 C:\Users\{user}\AppData\Roaming\SSHTunnelManager\config.json） |
| **损坏处理** | 启动时 JSON 解析失败 -> 尝试加载 config.json.bak -> 仍失败则提示用户，不覆盖原文件 |
| **备份策略** | 每次保存前写入 config.json.bak，保留上一版本 |
| **版本迁移** | version 字段标识格式版本，启动时检查并自动升级旧版配置 |
| **原子写入** | 先写临时文件 config.json.tmp，再 rename 覆盖，避免写入中途崩溃导致损坏 |

> 注：`AppSettings` 中的 `AutoStartMinimized`、`MinimizeToTrayOnClose`、`Theme` 字段会被持久化保存，但当前版本的 UI 代码并未读取或应用它们（主题 / 最小化到托盘功能尚未实现）。

### 4.4 安全性设计

| 要求 | 方案 |
|---|---|
| **IP 地址存储加密** | DPAPI ProtectedData.Protect() 加密。密钥绑定当前 Windows 用户 + 本机，config.json 泄露也无法解密 |
| **密码存储加密** | 同上，DPAPI 加密后存入 encryptedPassword 字段。运行时解密后仅存在于内存 |
| **界面显示脱敏** | 列表页 IP 显示为 123.45.***.***，密码永远不明文显示 |
| **Host Key 指纹** | 存储 SHA256 指纹，首次连接弹窗让用户确认，后续自动比对 |
| **SSH 私钥** | 仅存路径，不存私钥内容。安全由文件系统权限保证 |
| **日志安全** | 不记录密码、密钥内容、完整 IP（日志中用脱敏 IP） |

### 6.2 SSH.NET 核心代码结构

```csharp
using Renci.SshNet;

// 构造认证方式
AuthenticationMethod auth = config.AuthMethod switch
{
    AuthMethod.Password => new PasswordAuthenticationMethod(
        config.SshUser, CryptoHelper.Decrypt(config.EncryptedPassword)),
    AuthMethod.PrivateKey => new PrivateKeyAuthenticationMethod(
        config.SshUser, new PrivateKeyFile(config.KeyFilePath)),
};

var connInfo = new ConnectionInfo(host, config.SshPort, config.SshUser, auth);

var client = new SshClient(connInfo);

// Host Key 校验
client.HostKeyReceived += (sender, e) =>
{
    if (config.HostKeyTrust == HostKeyTrust.Trusted
        && e.FingerPrint == config.HostKeyFingerprint)
    {
        e.CanTrust = true;
    }
    else if (config.HostKeyTrust == HostKeyTrust.Unknown)
    {
        // 弹窗让用户确认，保存指纹
        e.CanTrust = ShowHostKeyDialog(e.FingerPrint, e.HostKeyName);
    }
    else
    {
        e.CanTrust = false;
    }
};

client.Connect();

// 反向端口转发（等同于 ssh -R）
var forward = new ForwardedPortRemote(
    "127.0.0.1", (uint)config.RemotePort,   // 远程机器监听
    "127.0.0.1", (uint)config.LocalPort);   // 转发到本机代理
client.AddForwardedPort(forward);
forward.Start();
```

### 6.3 连通性测试

隧道建立后，从本机通过 SSH.NET 的转发端口做一次验证。由于 ForwardedPortRemote 是远程机器监听，本机无法直接测试。两种方案：

**方案 A（推荐）**：通过 SSH 执行远程命令测试

```csharp
// 在远程机器上执行 curl，通过代理访问外网
var cmd = client.RunCommand(
    "curl -x http://127.0.0.1:" + config.RemotePort +
    " -s -o /dev/null -w '%{http_code}' https://www.google.com");
// 如果返回 200 或 301，隧道连通
```

**方案 B**：检查 ForwardedPort.IsStarted + SshClient.IsConnected

```csharp
if (forward.IsStarted && client.IsConnected)
    // SSH 连接活着且端口转发已启动，认为隧道可用
```

实际实现用方案 A 做首次验证，方案 B 做后续心跳。

### 6.4 自动重连策略

| 重连次数 | 等待时间 | 说明 |
|---|---|---|
| 1-3 次 | 3 秒 | 应对短暂网络抖动 |
| 4-5 次 | 10 秒 | 逐渐放慢频率 |
| 超过 5 次 | 停止 | 标记失败，由用户手动重试 |

### 6.5 应用退出处理

```
用户点击退出 / 关闭窗口
    |
    v
遍历所有运行中的隧道
    |
    v
对每个隧道：
  forward.Stop()     <- 停止端口转发
  client.Disconnect()  <- 断开 SSH 连接
    |
    v
保存当前配置状态
    |
    v
退出应用
```

## 7. 输入校验规则

| 字段 | 规则 | 错误提示 |
|---|---|---|
| 名称 | 非空，1-50 字符，同用户下唯一 | "名称不能为空" / "名称已存在" |
| SSH 用户 | 非空 | "SSH 用户不能为空" |
| SSH 地址 | 合法 IP 或域名格式 | "请输入合法的 IP 或域名" |
| SSH 端口 | 1-65535 整数 | "端口范围 1-65535" |
| 本机代理端口 | 1-65535 整数，默认 7890 | "端口范围 1-65535" |
| 远程映射端口 | 1-65535 整数，默认 1080 | "端口范围 1-65535" |
| 密码 | 密码认证时非空 | "密码不能为空" |
| 私钥文件 | 密钥认证时文件路径存在 | "私钥文件不存在" |
| 远程映射端口唯一性 | 同一台远程机器（相同 IP+SSH端口）不可重复 | "该机器已有隧道使用此远程端口" |

```csharp
    _mutex = new Mutex(true, "SSHTunnelManager_SingleInstance", out bool createdNew);
```

## 11. 风险与注意事项

### 11.1 前置依赖

- 本机需有代理客户端独立运行（Clash/v2ray 等），监听在配置的本地端口
- 远程机器需运行 SSH Server
- 远程机器的 GatewayPorts no（默认值）无需改动
- 不再依赖系统 OpenSSH（SSH.NET 是纯 .NET 实现）

### 11.2 异常处理

| 场景 | 处理方式 |
|---|---|
| SSH 连接超时 | 10 秒超时，提示"无法连接到远程服务器" |
| SSH 认证失败 | 捕获 SshAuthenticationException，提示检查用户名/密码/密钥 |
| Host Key 不匹配 | 拦截连接，提示"服务器指纹已变更，可能存在安全风险" |
| 端口转发失败（远程端口被占） | 捕获异常，提示"远程端口 XXX 已被占用" |
| 本地代理端口未监听 | 启动前检测，警告但允许继续 |
| 网络中断 | 自动重连（5 次上限） |
| DPAPI 解密失败 | 温和提示，不崩溃 |
| config.json 损坏 | 加载 .bak 备份，仍失败则提示用户 |

### 11.3 已知限制

- 不支持多跳隧道（仅单层反向隧道）
- 不内置代理 Client（需用户自行运行 Clash/v2ray 等）
- 密钥认证仅支持 OpenSSH 格式私钥（id_rsa、id_ed25519）
- 不支持 SSH Agent 转发
